## 7.30.0 {#7_30_0}

### Изменения в XML API {#changes-xml-api}

- Удалены компоненты `<left-right-page>` и `<top-left-right-page>`.
- Удален способ передачи в open-page `:key` прим. `<path-param name="id" value=":id"/>`
  Необходимо использовать стандартный формат `{key}` прим. `<path-param name="id" value="{id}" />`
- Удален компонент `<calendar>`.
- Полностью прекращена поддержка старых схем: `query-4.0`, `application-2.0`, `page-3.0`, `n2o-widget-4.0`, `region-2.0`,
`fieldset-4.0`, `toolbar-base-1.0`, `button-1.0`, `action-1.0`, `control-2.0`, `n2o-cell-2.0`, `properties-base-1.0`.

### Изменения поведения {#changes-behavior}

- `<fetch>` и `<fetch-value>` зависимости полей теперь не выполняются, если поле или компонент,
в котором находится поле (филдсет, форма), скрыты.

### Изменения в сборке {#changes-build}

- Движок выполнения JS выражений заменён с Rhino + Nashorn на **GraalJS** (GraalVM 23.0.5).

  Зависимости `org.mozilla:rhino` и `org.openjdk.nashorn:nashorn-core` удалены из classpath фреймворка.
  Если в вашем проекте они использовались напрямую, их необходимо добавить в `pom.xml` явно.

### Расширенная поддержка JS в XML {#changes-js-expressions}

Ранее JS выражения в XML файлах исполнялись движком Nashorn, который поддерживал только стандарт ES5.
Начиная с версии 7.30 используется GraalJS с поддержкой **ECMAScript 2020**. О том, какие возможности доступны
можете прочитать в [официальной документации](https://www.graalvm.org/latest/reference-manual/js/JavaScriptCompatibility/).
